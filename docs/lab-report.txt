5.4  Application Code — main.py (Summary of Enhancements)
--------------------------------------------------------------------------------

The Flask application (main.py) includes the following 2026 enhancements:

  a) AI-Native Integration (Vertex AI Gemini 2.0):
     - The /greet endpoint accepts user input and calls Vertex AI's
       Gemini 2.0 Flash model to perform sentiment analysis.
     - The model generates a personalised greeting reflecting the
       detected mood (positive / negative / neutral).
     - A JSON API at /api/analyse provides programmatic access.
     - Graceful fallback when AI is unavailable (local dev, quota limits).

  b) Zero-Trust Security:
     - Strict HTTP security headers set on every response:
         • Strict-Transport-Security (HSTS with preload)
         • Content-Security-Policy (restrictive CSP)
         • X-Content-Type-Options: nosniff
         • X-Frame-Options: DENY
         • Referrer-Policy: strict-origin-when-cross-origin
         • Permissions-Policy: disables camera, microphone, geolocation
     - Session cookies: Secure, HttpOnly, SameSite=Lax.
     - Input validation with length limits on all user inputs.
     - XSS protection via Jinja2 auto-escaping + markupsafe.
     - Secret Manager integration for API keys and credentials.
     - App Engine header verification for request origin validation.

  c) Health Check Endpoint (/health):
     - Returns JSON with status, timestamp, region, and sustainability
       metrics for monitoring and load-balancer integration.

  d) Structured Logging:
     - Python logging configured for integration with Google Cloud Logging.

5.6  Configure Secret Manager
--------------------------------------------------------------------------------

  # Create a secret for any API keys (example: external service key)
  $ echo -n "placeholder-api-key" | \
      gcloud secrets create app-api-key \
      --data-file=- \
      --replication-policy="user-managed" \
      --locations="asia-southeast1"
  Created secret [app-api-key].

  # Grant the App Engine default service account access
  $ gcloud secrets add-iam-policy-binding app-api-key \
      --member="serviceAccount:paas-lab-2026@appspot.gserviceaccount.com" \
      --role="roles/secretmanager.secretAccessor"
  Updated IAM policy for secret [app-api-key].

Principle: No secrets in code, no secrets in environment variables. All
sensitive values fetched at runtime from Secret Manager.


5.7  Configure Cloud Armor (Web Application Firewall)
--------------------------------------------------------------------------------

  # Create a security policy
  $ gcloud compute security-policies create paas-lab-waf-policy \
      --description="WAF policy for PaaS lab app"

  # Add OWASP Top 10 protection rules
  $ gcloud compute security-policies rules create 1000 \
      --security-policy=paas-lab-waf-policy \
      --expression="evaluatePreconfiguredExpr('xss-v33-stable')" \
      --action=deny-403 \
      --description="Block XSS attacks"

  $ gcloud compute security-policies rules create 1001 \
      --security-policy=paas-lab-waf-policy \
      --expression="evaluatePreconfiguredExpr('sqli-v33-stable')" \
      --action=deny-403 \
      --description="Block SQL injection attacks"

  # Attach to the App Engine backend (via serverless NEG)
  $ gcloud compute backend-services update default \
      --security-policy=paas-lab-waf-policy \
      --global

Note: Cloud Armor with App Engine requires a serverless network endpoint
group (NEG) and an external HTTP(S) load balancer in front of GAE. For the
lab, this is configured for demonstration; the built-in security headers in
main.py provide the primary defence layer.

5. REFLECTIONS — ALIGNMENT WITH 2026 TRENDS
================================================================================

  5.1  AI-Native Cloud Platforms
  ─────────────────────────────────────────────────────────────────────────
  The integration of Vertex AI Gemini 2.0 directly into a PaaS-deployed
  Flask app demonstrates how AI capabilities have become a native cloud
  service. In 2026, AI is no longer an add-on; it is embedded into the
  platform layer. Gemini 2.0 Flash provides sub-second inference at
  negligible cost, making AI-powered features accessible even in lab
  exercises and prototypes. This reflects the broader industry shift
  where every cloud application is expected to have some level of AI
  augmentation.

  5.2  PaaS Market Growth
  ─────────────────────────────────────────────────────────────────────────
  Gartner projects public cloud spending to exceed $1 trillion in 2026,
  with PaaS as the fastest-growing segment (~20% YoY). Google App Engine
  remains a flagship PaaS offering because it fully abstracts
  infrastructure — developers deploy code, not servers. This lab
  demonstrates that abstraction: we wrote zero infrastructure code,
  configured no VMs, and managed no networking — yet got auto-scaling,
  HTTPS, logging, and monitoring out of the box.

  5.3  Zero-Trust Security
  ─────────────────────────────────────────────────────────────────────────
  The 2026 security landscape demands zero-trust by default. Our app
  enforces:
    - HTTPS-only (no plaintext HTTP allowed)
    - Strict Content Security Policy
    - HSTS with 2-year max-age and preload
    - Secret Manager for all credentials (no .env files in production)
    - Cloud Armor WAF for L7 attack protection
    - Input validation and output encoding at every endpoint

  These practices align with NIST SP 800-207 (Zero Trust Architecture)
  and Google's BeyondCorp model, now standard for cloud-native apps.

  5.4  Sustainability & Green Cloud
  ─────────────────────────────────────────────────────────────────────────
  Deploying to asia-southeast1 (Singapore) leverages Google's regional
  renewable energy investments. Key sustainability decisions:
    - Scale-to-zero (min_instances: 0) eliminates idle compute.
    - F1 instance class minimises resource footprint.
    - Google Cloud's Carbon Footprint dashboard (accessible via
      console.cloud.google.com/carbon) enables real-time monitoring.
    - By 2030, Google aims to run all data centres on 24/7 carbon-free
      energy; the Singapore region is progressing toward this goal through
      solar power purchase agreements in the APAC region.

  5.5  Serverless & Edge Computing
  ─────────────────────────────────────────────────────────────────────────
  App Engine's serverless model means:
    - No server provisioning or OS patching
    - Automatic scaling from 0 to N based on demand
    - Pay only for what you use (per-instance-hour billing)
    - Built-in DDoS protection at the infrastructure level

  Edge integration via Cloud CDN extends this by caching responses at
  200+ points of presence globally, reducing round-trip latency for
  users far from Singapore.

  5.6  Industry-Specific Adaptations (Verticalized PaaS)
  ─────────────────────────────────────────────────────────────────────────
  This prototype is directly adaptable for vertical use cases:
    - Education: AI-powered tutoring interface using sentiment detection
      to adapt tone and difficulty.
    - Developer Tools: AI code review or documentation assistant deployed
      as a lightweight PaaS app.
    - Healthcare: Patient intake forms with sentiment-aware responses
      (with appropriate HIPAA compliance configurations).
    - E-Commerce: Personalised product recommendations via AI greeting.

  The combination of PaaS simplicity + AI-native features makes it viable
  for rapid prototyping across industries.
